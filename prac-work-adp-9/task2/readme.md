# Курс: Шаблоны проектирования приложений

## Тема: Модуль 09 Структурные паттерны.Компоновщик.
 
 # Corporate Hierarchy Management System

## Цель
Изучить и применить паттерн **Компоновщик (Composite)** для построения системы, моделирующей древовидную иерархию компонентов. В качестве примера используется корпоративная структура организации, где каждый элемент может быть как отдельным сотрудником, так и группой объектов (подразделений).

---

## Описание задачи
Создать систему управления корпоративной иерархией организации. В системе:

- Отдельные сотрудники и подразделения представлены как компоненты с единым интерфейсом.
- Каждый отдел может содержать сотрудников и другие подчиненные отделы, образуя древовидную структуру.
- Система позволяет:
  - Добавлять сотрудников и подразделения.
  - Отображать структуру организации с иерархией.
  - Рассчитывать общий бюджет отдела (с учетом зарплат сотрудников).
  - Рассчитывать общий штат сотрудников в отделе и его подчиненных подразделениях.

---

## Требования

1. **Абстрактный класс `OrganizationComponent`**  
   Представляет как сотрудников, так и подразделения.
   
2. **Класс `Employee`**  
   - Имя сотрудника (`Name`)  
   - Должность (`Position`)  
   - Зарплата (`Salary`)  
   
3. **Класс `Department`**  
   - Может содержать сотрудников (`Employee`) и другие отделы (`Department`).  
   
4. **Функциональность**  
   - Расчет общего бюджета подразделения.  
   - Подсчет общего количества сотрудников.  
   - Отображение структуры организации.

---

## Дополнительные задания

1. Возможность изменения зарплаты сотрудников и пересчета бюджета.  
2. Поиск сотрудника по имени и отображение его информации.  
3. Поддержка **контракторов** (временные сотрудники с фиксированной оплатой), зарплата которых не включается в бюджет.  
4. Метод для отображения всех сотрудников отдела и его подчиненных отделов в виде списка.
